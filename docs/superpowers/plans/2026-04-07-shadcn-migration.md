# shadcn/ui Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace custom Tailwind UI elements with shadcn/ui components across the entire Fenua Chords MVP while preserving the "Lagon & Terre" design system.

**Architecture:** Install shadcn via CLI with Tailwind v4 support, map our custom CSS variables to shadcn's theming system, then migrate file by file — starting with base components (Button, Input) and working up to complex ones (Sheet, Command, Tabs). Each task produces a buildable state.

**Tech Stack:** shadcn/ui, Tailwind CSS v4, Next.js 16, React 19, TypeScript strict, lucide-react.

**Spec:** `docs/superpowers/specs/2026-04-07-shadcn-migration-design.md`

Replace custom input with shadcn Input:

```tsx
// src/components/SearchBar.tsx
"use client";

import { useRouter, useSearchParams } from "next/navigation";
import { useState } from "react";
import { Search } from "lucide-react";
import { Input } from "@/components/ui/input";

export function SearchBar() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const [query, setQuery] = useState(searchParams.get("q") ?? "");

  function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    const params = new URLSearchParams(searchParams.toString());
    if (query.trim()) {
      params.set("q", query.trim());
    } else {
      params.delete("q");
    }
    router.push(`/chansons?${params.toString()}`);
  }

  return (
    <form onSubmit={handleSubmit} className="relative">
      <Search className="absolute left-3 top-1/2 -translate-y-1/2 w-4 h-4 text-muted-foreground" />
      <Input
        type="search"
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Chercher un titre ou un artiste..."
        className="pl-10"
      />
    </form>
  );
}
```
